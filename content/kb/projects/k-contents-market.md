# K-Contents IP Market

K-콘텐츠 IP 거래 플랫폼

## 소개

K콘텐츠의 온라인 IP 거래를 위한 B2B 서비스 신규 프로젝트에 프론트엔드 전담 개발자로 참여하여 플랫폼 및 CMS의 프론트엔드를 개발하였습니다. 개발 과정에서 기획자와 디자이너, 백엔드 개발자와 면밀히 소통하였으며 아키텍처, 성능 목표를 비롯한 모든 프론트엔드 시스템 설계를 전담하였습니다.

- **형태**: 사내 프로젝트(올콘텐츠앤에이아이)
- **참여 역할**: 프론트엔드 설계 및 개발
- **주요 기술**: Next.js, Tanstack Query, React-Admin, Docusaurus

## 경험

### 플랫폼

Next.js 기반 반응형 웹 페이지로 구현하였습니다. 기획과 UI/UX 요구 사항을 모두 충족하면서도 성능 목표 달성과 객체 지향 설계를 위해 노력하였습니다.

- Tailwind CSS 및 Headless UI 적극 활용하여 모달, 드롭다운과 같은 다양한 컴포넌트를 빠르게 구현
- Next.js Middleware를 활용한 다국어 지원 및 Dictionary 체계 구현
- Swiper.js 대응을 위한 SSR Placeholder를 포함, 스켈레톤 UI를 구현하여 0.1 이하 수준의 CLS를 확보
- 페이지 특성을 고려하여 메인/검색 페이지 SSR 적용, 이를 통해 FCP 향상 및 초기 렌더링에 자사 콘텐츠 노출하여 SEO 확보
- TanStack Query를 활용하여 콘텐츠 정보와 같은 주요 API 응답 데이터 캐시 및 Optimistic Update 구현으로 페이지 로드 및 인터랙션 성능 확보
- Schema Driven Form을 자체 구현하여 방대한 카테고리 및 장르에 따른 폼 레이아웃 변화에 유연하게 대응함과 동시에 확장성 확보
- 객체 지향적인 모듈 경계 수립을 위해 FSD 아키텍처 도입, 변경 용이성 확보

### CMS

React Admin을 활용하여 빠르고 안정적인 CMS를 구현하였습니다. 짧은 기간 안에 완성도 높은 시스템을 개발해야 했기 때문에 프레임워크에서 제공하는 기본 기능을 적극 탐색 및 활용하였습니다.

- Material UI 커스터마이징으로 디자인 요구 사항을 빠르게 대응
- API 응답 데이터에 대한 캐시 및 무효화 전략 수립하여 성능과 데이터 일관성 사이에서 트레이드 오프 조율

### 운영

- Sentry 연동으로 QA 및 프로덕션 환경에서 신속하게 에러 추적 및 장애 대응
- Docker 및 GitHub Actions 활용하여 EC2 인스턴스 상에서의 이미지 로드 자동화로 CD 파이프라인 구축, 배포 안정성 향상

### 문서화

Docusaurus를 활용하여 개발 가이드 문서를 작성하였습니다. 지속적인 유지보수를 염두에 두어 단순한 파일 형태의 문서가 아닌 정적 페이지 형태로 사내에 공유하였습니다.

- Swiper 커스터마이징, SSR Placeholder, Schema Driven Form 등 독자적이라고 생각되는 노하우를 전수함으로써 팀 생산성 및 사내 개발 문화에 기여
- 설계 철학과 컨벤션, 레퍼런스를 상세히 기술하여 코드베이스 일관성 확보
- 변경 사항을 기록하는 별도의 섹션을 만들어 프로젝트 히스토리 보존에 기여
